## CSS Architecture Summary

### Design Tokens (lines 29-51, 388-415)
- Two `:root` blocks: website tokens (`--w-*`) and dashboard tokens (`--bg-*`, `--text-*`, `--accent-*`)
- Dark mode is default

### Light Mode (lines 1848-2782)
- Activated via `[data-theme="light"]` on `<html>`
- Overrides all CSS custom properties + additional selectors to override inline styles in JSX
- Currently forced to dark mode by default (light mode WIP ‚Äî see git log)

### Responsive Breakpoints
- **1024px** (tablet): Nav padding, 4‚Üí2 col grids, stats sizing (line 832)
- **768px** (mobile): Full mobile layout ‚Äî nav hamburger, 1-col grids, section padding, dashboard sidebar collapse, code block overflow (line 1183)
- **480px** (small mobile): Heading size caps, full-width buttons, 1-col everything, trust badge stacking (line 1458)
- **Hero section**: Mobile-first (base = mobile, min-width 768px = desktop) (line 866)

- `.landing-hero`, `.hero-content`, `.hero-heading`, `.hero-btn-primary` ‚Äî Hero section
